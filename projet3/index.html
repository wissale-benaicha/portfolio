Le Perchoir

Site de gestion d’événements avec réservations codé sous PHP et MySQL

Contexte

Dans le cadre de la SAE 203 en première année de MMI, j'ai du créer un site web dynamique permettant à des utilisateurs de consulter et réserver des événements, avec un système de validation de compte par code.

Démarche

Présentation

J’ai développé un site de gestion d’événements où un utilisateur peut s’inscrire, se connecter, réserver des événements, et voir ceux auxquels il est inscrit. Le site propose également une interface d’administration qui permet d’ajouter, modifier ou supprimer des événements.

Pour sécuriser les inscriptions, un code de validation est envoyé à l’utilisateur après son inscription. Il doit ensuite le saisir sur une page dédiée pour que son compte soit activé..

Fonctionnalités principales

  • Inscription avec vérification par code : Lorsqu’un utilisateur s’inscrit, un code à 5 chiffres lui est généré et affiché (ou envoyé). Il doit ensuite le rentrer sur une page de vérification pour activer son compte.

  • Connexion et inscription : Lorsqu’un utilisateur s’inscrit, un code à 5 chiffres lui est généré et affiché (ou envoyé). Il doit ensuite le rentrer sur une page de vérification pour activer son compte.

  • Réservation d’événements : Une fois connecté, il peut réserver une place pour les événements encore disponibles.

  • Espace admin : Un administrateur peut gérer les événements (ajout, modification, suppression) et consulter les réservations.

    • Conception de la base de données et démarche de réalisation

    Avant de coder, j’ai commencé par concevoir un Modèle Conceptuel de Données, puis un Modèle Logique de Données pour définir les différentes tables (utilisateurs, événements, réservations, catégories, etc.) et leurs relations. Cela m’a permis d’organiser ma base de données MySQL de manière cohérente.

    La MCD

    La MLD

    J’ai ensuite créé les différentes pages du site en PHP (inscription, connexion, liste des événements, formulaire de réservation…). J’ai utilisé les sessions PHP pour gérer l’authentification, la validation de compte et les messages d’erreur.

    Pour l’enregistrement des événements et la réservation, j’ai utilisé des requêtes préparées avec PDO pour sécuriser les échanges avec la base de données.

    Problèmes rencontrés

    Au début, j’ai eu du mal avec la gestion des sessions entre plusieurs pages. Par exemple, lors de la vérification du code, il fallait bien récupérer le bon nom d’utilisateur pour mettre à jour son statut dans la base.

    J’ai aussi eu quelques difficultés avec la mise à jour des données et les redirections après la validation. Il fallait faire attention à ce que les données restent disponibles entre les pages, soit via SESSION, soit via l’URL via GET.

    Mais, avec un peu de temps et un peu d'effort, j'ai réussi à faire un site qui marche. Simple, mais fonctionnel.

    Résultat et ce que j'ai appris

    Le site fonctionne correctement : un utilisateur peut s’inscrire, valider son compte, se connecter, consulter les événements disponibles et en réserver. De son côté, l’administrateur peut gérer les événements via une interface simple. Le tout repose sur une base de données relationnelle bien structurée.

    Ce projet m’a permis de mieux comprendre le lien entre la base de données et le code PHP, mais aussi l’importance des sessions et des vérifications de sécurité.
